## 🚀 立即開始（3 個簡單步驟）

### 步驟 1️⃣：創建 Supabase 項目（2 分鐘）

```
1. 打開瀏覽器：https://supabase.com
2. 用 Google 或 GitHub 快速登入
3. 點「New Project」
4. 填入名稱和密碼，選擇地區
5. 等待 1-2 分鐘，項目創建完成
```

### 步驟 2️⃣：複製你的憑證（1 分鐘）

```
1. 進入你的 Supabase 項目
2. 左側菜單 → Settings → API
3. 複製 2 個信息：
   ✓ Project URL（長鏈接）
   ✓ anon public（長字符串）
4. 暫時保存到記事本
```

### 步驟 3️⃣：在應用中配置（1 分鐘）

```
1. 打開 index.html
2. 向下滾動找到「數據庫配置」卡片
3. 貼入：URL 和 Key
4. 點「連接數據庫」按鈕
5. 看到 ✓ 綠色提示 = 完成！
```

## 🤔 常見問題

### Q：配置無法連接？
A：檢查：
- URL 和 Key 是否有多餘空格
- URL 格式是否為 `https://...supabase.co`
- Supabase 項目是否真的存在

### Q：數據沒有保存？
A：檢查：
- 頁面頂部是否顯示「已連接 ✓」
- F12 Console 是否看到 ✓ 成功信息
- 在 Supabase 刷新頁面重新查看

### Q：配置為什麼會消失？
A：不會！配置自動保存在瀏覽器。如果消失：
- 清除了瀏覽器 Cookies
- 使用了無痕/隱私模式
- 或硬刷頁面（重新進入會自動恢復）

### Q：多個用戶使用同一個應用？
A：完全沒問題！所有記錄都用時間戳分開，可以輕鬆區分。

### Q：我想改變配置？
A：直接重新輸入 URL/Key 點「連接」，系統會自動更新。

## ⚡ 最常用的 3 個操作

### 操作 1：查看最新記錄
```sql
SELECT * FROM decision_logs 
ORDER BY created_at DESC 
LIMIT 10;
```

### 操作 2：統計不同方案的使用頻率
```sql
SELECT 
  solution_label,
  COUNT(*) as 使用次數
FROM decision_logs
GROUP BY solution_label;
```

### 操作 3：分析不同製程節點的平均成本
```sql
SELECT 
  node,
  AVG(solution_cost) as 平均成本
FROM decision_logs
GROUP BY node;
```
